# 代码审查技能

> 改编自 Everything Claude Code 的 code-reviewer agent。

## 审查清单

### 1. 正确性
- [ ] 逻辑是否正确？
- [ ] 边界条件是否处理？
- [ ] 错误路径是否覆盖？
- [ ] 类型是否正确？

### 2. 安全性
- [ ] 无硬编码密钥/token
- [ ] 用户输入已验证
- [ ] 无 XSS 风险（避免 v-html）
- [ ] API 调用有认证
- [ ] 敏感数据不暴露在前端

### 3. 性能
- [ ] 无不必要的重渲染
- [ ] 大列表使用虚拟滚动
- [ ] 图片/资源已优化
- [ ] API 调用有缓存策略
- [ ] 无内存泄漏（清理定时器/监听器）

### 4. 可维护性
- [ ] 命名清晰有意义
- [ ] 函数职责单一
- [ ] 无重复代码
- [ ] 复杂逻辑有注释
- [ ] 类型定义完整

### 5. Vue 特定
- [ ] 使用 Composition API + `<script setup>`
- [ ] Props 有类型定义
- [ ] 响应式数据使用正确（ref vs reactive vs shallowRef）
- [ ] 组件拆分合理（不超过 300 行）
- [ ] 生命周期钩子中清理副作用

## 审查流程

1. 先看整体变更范围和目的
2. 检查架构/设计是否合理
3. 逐文件审查细节
4. 运行测试确认通过
5. 提出改进建议（区分必须修改 vs 建议优化）

## 反馈格式

```
🔴 必须修改：[描述问题和原因]
🟡 建议优化：[描述改进方向]
🟢 做得好：[值得肯定的地方]
```
