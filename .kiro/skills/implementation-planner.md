# 实现规划技能

> 改编自 Everything Claude Code 的 planner agent。

## 何时使用

- 开始一个新功能开发
- 进行大规模重构
- 解决复杂 bug
- 架构变更

## 规划流程

### 1. 需求分析

```markdown
## 需求
- 用户故事：作为 [角色]，我想要 [功能]，以便 [价值]
- 验收标准：
  - [ ] 标准 1
  - [ ] 标准 2
- 约束条件：[技术/时间/资源限制]
```

### 2. 影响分析

- 哪些文件需要修改？
- 哪些模块会受影响？
- 是否需要数据库变更？
- 是否需要 API 变更？
- 是否有向后兼容性问题？

### 3. 实现方案

```markdown
## 方案

### 方案 A: [名称]
- 优点：
- 缺点：
- 工作量：[小/中/大]
- 风险：[低/中/高]

### 方案 B: [名称]
- 优点：
- 缺点：
- 工作量：
- 风险：

### 选择：方案 [X]
理由：[为什么选这个]
```

### 4. 任务分解

将实现拆分为可独立完成和验证的小任务：

```markdown
## 任务列表

### Phase 1: 基础设施
- [ ] 任务 1.1: [描述] (~X min)
- [ ] 任务 1.2: [描述] (~X min)

### Phase 2: 核心实现
- [ ] 任务 2.1: [描述] (~X min)

### Phase 3: 测试和验证
- [ ] 任务 3.1: 编写单元测试
- [ ] 任务 3.2: 集成测试
- [ ] 任务 3.3: 验证检查点
```

### 5. 风险评估

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| [风险1] | 高/中/低 | 高/中/低 | [措施] |
