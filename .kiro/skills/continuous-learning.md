# 持续学习

> 改编自 Everything Claude Code 的 continuous-learning skill。

## 概念

从每次开发会话中自动提取可复用的模式和经验，形成项目特定的知识库。

## 学习流程

```
开发会话 → 识别模式 → 提取经验 → 保存到 skills → 下次复用
```

## 何时提取模式

- 解决了一个棘手的 bug
- 发现了更好的实现方式
- 踩了一个坑并找到了解决方案
- 建立了新的架构模式
- 优化了性能

## 模式记录格式

```markdown
## 模式: [名称]

### 场景
[什么情况下会遇到这个问题]

### 问题
[具体的问题描述]

### 解决方案
[解决方案和代码示例]

### 注意事项
[使用时需要注意的点]

### 置信度
[高/中/低] - 基于验证次数
```

## 项目特定模式库

将提取的模式保存到 `.kiro/skills/` 目录下，按领域分类：

```
.kiro/skills/
├── patterns-api.md        # API 调用模式
├── patterns-vue.md        # Vue 组件模式
├── patterns-state.md      # 状态管理模式
├── patterns-testing.md    # 测试模式
└── patterns-debug.md      # 调试技巧
```

## 模式演化

- 新模式：置信度 = 低
- 验证 2 次后：置信度 = 中
- 验证 5 次后：置信度 = 高
- 高置信度模式可以提升为 steering 规则

## 会话结束时

在每次较长的开发会话结束时，回顾：

1. 这次会话学到了什么？
2. 有哪些可复用的模式？
3. 有哪些需要避免的坑？
4. 更新相关的 skill 文件
