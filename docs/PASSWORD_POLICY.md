# 密码策略变更说明（前端配合）

## 变更概述

后端已升级注册和密码重置的密码策略。**仅影响新注册用户和密码重置场景，老用户登录不受影响。**

## 新密码要求

| 规则 | 旧策略 | 新策略 |
|------|--------|--------|
| 最小长度 | 6 字符 | **12 字符** |
| 最大长度 | 20 字符 | 128 字符 |
| 大写字母 | 需要 | 需要 |
| 小写字母 | 需要 | 需要 |
| 数字 | 需要 | 需要 |
| 特殊字符 | 需要 | 需要 |
| 弱密码检查 | 无 | **新增** |

## 影响范围

| 场景 | 是否受影响 |
|------|-----------|
| 新用户注册 | ✅ 必须满足新策略 |
| 密码重置 | ✅ 必须满足新策略 |
| 老用户登录 | ❌ 不受影响，原密码可继续使用 |

## 前端需要配合的改动

### 1. 注册页面

- 密码输入框的最小长度提示改为 12 字符
- 最大长度放宽到 128 字符
- 建议添加实时密码强度提示，规则：
  - ≥12 字符
  - 包含大写字母 (A-Z)
  - 包含小写字母 (a-z)
  - 包含数字 (0-9)
  - 包含特殊字符 (!@#$%^&* 等)

### 2. 密码重置页面

- 同注册页面的密码规则

### 3. 登录页面

- **无需改动**，老用户仍可使用原密码登录

## 后端错误响应示例

密码不符合策略时，后端会返回具体的错误信息数组：

```json
{
  "password": [
    "密码长度不能少于 12 个字符",
    "密码必须包含至少一个大写字母",
    "密码必须包含至少一个特殊字符"
  ]
}
```

前端可以直接展示这些错误信息给用户。

## 密码策略描述文案（可直接使用）

> 密码要求：至少 12 个字符，包含大写字母、小写字母、数字和特殊字符
